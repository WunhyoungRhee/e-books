CHAPTER 13

문제 해결 (Troubleshooting)

안 되는 건 당연해요. 해결하면 되니까요

에러가 나면 축하드려요

농담이 아니에요. 에러가 나면 축하할 일이에요. 왜냐하면 뭔가를 시도했다는 뜻이니까요.

아무것도 안 하면 에러도 안 나요. 에러가 났다는 건 여러분이 용감하게 도전했다는 증거예요. 그러니까 당황하지 마시고, 이 챕터를 천천히 따라오세요.

쉬운 비유

자전거를 배울 때 넘어지지 않는 사람은 없어요. 중요한 건 넘어진 다음에 다시 타는 거예요. 에러도 마찬가지. 만나고, 해결하고, 다음에는 같은 실수를 안 하면 돼요.

초보자가 가장 많이 겪는 문제 3가지

문제 1: "이거 ChatGPT랑 뭐가 다른 거예요?"

이건 에러가 아니라 혼란이에요. 근데 아주 자연스러운 혼란이에요.

ChatGPT는 웹 브라우저에서 대화하는 챗봇이에요. 질문하면 답해주죠. gstack(Claude Code)은 터미널에서 작동하는 AI 에이전트예요. 질문에 답하는 것뿐 아니라, 파일을 만들고, 코드를 실행하고, 웹사이트를 테스트하고, 프로젝트를 관리해요.

핵심 차이

ChatGPT: "이 코드가 뭘 하는지 설명해줘" → 설명을 텍스트로 줌
gstack: "이 프로젝트의 버그를 찾아서 고쳐줘" → 실제로 파일을 열어서 코드를 수정함

채팅 vs 실제 작업. 이게 가장 큰 차이예요.

문제 2: "터미널이 뭔지 모르겠어요"

터미널이 무서워 보이는 건 정상이에요. 까만 화면에 글자만 있으니까요. 근데 사실 터미널은 컴퓨터에게 글로 말 거는 방법이에요.

평소에는 마우스로 폴더를 클릭하고, 파일을 더블클릭하잖아요? 터미널에서는 같은 걸 글자로 해요.

  • 폴더 열기 (마우스 더블클릭) → cd 폴더이름
  • 파일 목록 보기 (탐색기 열기) → ls
  • 프로그램 실행 (아이콘 더블클릭) → 프로그램 이름 입력

그게 다예요. 무서울 거 하나도 없어요. 처음엔 어색하지만, 며칠 쓰면 마우스보다 편해져요.

터미널 여는 법

Windows: 시작 메뉴에서 "PowerShell" 또는 "명령 프롬프트" 검색
Mac: Command + Space → "터미널" 검색
Linux: Ctrl + Alt + T

문제 3: "슬래시 명령어가 작동 안 해요"

/browse, /qa 같은 슬래시 명령어를 쳤는데 아무 반응이 없다면, 몇 가지를 확인해보세요:

  1. gstack 안에서 치고 있나요? 슬래시 명령어는 gstack 세션 안에서만 작동해요. 일반 터미널에서 치면 안 돼요. 먼저 claude를 입력해서 gstack을 시작하세요.
  2. 스킬이 설치되어 있나요? 슬래시 명령어는 해당 스킬의 SKILL.md 파일이 있어야 작동해요. /find-skills로 설치된 스킬 목록을 확인하세요.
  3. 오타가 있지 않나요? /Browse(대문자)와 /browse(소문자)는 다를 수 있어요. 소문자로 정확히 입력하세요.

기술적 문제 해결

문제 4: 설치가 안 돼요

가장 흔한 원인은 Node.js가 설치되지 않았거나 버전이 오래된 경우예요.

  • node --version을 터미널에서 입력해보세요. 버전 번호가 나오면 설치된 거예요.
  • "command not found"가 나오면 Node.js를 먼저 설치하세요: nodejs.org
  • Node.js 18 이상이 필요해요. 버전이 낮으면 업데이트하세요.

문제 5: API 키 오류

"Invalid API key" 또는 "Authentication failed" 에러가 나온다면:

  • API 키를 복사할 때 앞뒤 공백이 포함되지 않았는지 확인하세요
  • API 키가 만료되지 않았는지 확인하세요
  • 환경 변수가 제대로 설정되었는지 확인하세요

보안 주의!

API 키는 절대 다른 사람에게 보여주거나 공유하지 마세요. 채팅, 이메일, 스크린샷에 API 키가 포함되지 않도록 주의하세요. API 키가 노출되면 즉시 재발급받으세요.

문제 6: 응답이 너무 느려요

AI가 응답하는 데 시간이 오래 걸린다면:

  • 인터넷 연결 상태를 확인하세요
  • 요청이 너무 복잡하지 않은지 확인하세요 — 큰 작업은 작은 단계로 나눠보세요
  • 서버가 혼잡한 시간대(미국 업무 시간)를 피해보세요

문제 7: AI가 엉뚱한 결과를 줘요

이건 대부분 프롬프트가 애매한 경우예요.

  • 요청을 더 구체적으로 바꿔보세요
  • 원하는 결과의 예시를 함께 제공하세요
  • "이건 아니야, 이런 식으로 해줘"라고 피드백을 주세요 — AI는 대화를 통해 점점 정확해져요

문제 8: 파일이 의도치 않게 변경됐어요

gstack은 파일을 직접 수정할 수 있는 강력한 도구예요. 실수가 생길 수 있어요.

  • Git을 쓰고 있다면: git diff로 변경사항을 확인하고, git checkout -- 파일이름으로 되돌릴 수 있어요
  • Git을 안 쓰고 있다면: 다음부터는 중요한 작업 전에 /careful을 실행하세요
  • 예방이 최선이에요: /freeze로 작업 디렉토리를 제한하면 다른 폴더의 파일은 안전해요

문제 9: 브라우저 자동화(/browse)가 안 돼요

헤드리스 브라우저가 제대로 작동하지 않는 경우:

  • Chromium 기반 브라우저가 설치되어 있는지 확인하세요
  • 방화벽이나 보안 소프트웨어가 브라우저 실행을 차단하고 있지 않은지 확인하세요
  • 메모리가 부족할 수 있어요 — 다른 프로그램을 좀 닫아보세요

여기서 해결 안 되면?

GitHub Issues에 문의하세요

이 챕터에서 다루지 못한 문제가 있다면, GitHub Issues에 올려주세요. 커뮤니티와 개발팀이 도와줄 거예요.

이슈를 올릴 때 포함할 것:

  • 운영체제 (Windows/Mac/Linux)
  • Node.js 버전
  • gstack 버전
  • 에러 메시지 전체 (스크린샷 또는 복사)
  • 어떤 상황에서 발생했는지 재현 단계

정보가 자세할수록 빠르게 도움받을 수 있어요.

그리고 기억하세요: 문제가 생기는 건 여러분 잘못이 아니에요. 소프트웨어에는 항상 예상치 못한 상황이 있어요. 중요한 건 포기하지 않는 것이에요. 모든 전문가도 처음엔 이런 문제들을 겪었어요.
