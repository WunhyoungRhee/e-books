준비하기: 설치 전에 알아야 할 것들

처음 보는 도구들, 겁먹지 마세요

새 직장 첫 날의 복합기

새 직장 첫 날, 복합기 사용법 모르는 거 당연하잖아요? "이 버튼이 복사고, 이건 스캔이고, 양면 인쇄는 여기서..." 선배가 알려주면 5분이면 배워요.

터미널도 마찬가지예요. 처음 보면 까만 화면에 깜빡이는 커서가 무섭게 느껴질 수 있어요. 하지만 사실은 복합기만큼이나 단순한 도구예요. 이 챕터에서 천천히 알려드릴게요.

gstack을 설치하기 전에 알아야 할 도구가 4개 있어요. 하나씩 가볍게 만나봅시다.

1. 터미널(Terminal): 컴퓨터와 대화하는 창

여러분은 평소 컴퓨터를 어떻게 쓰나요? 아이콘을 클릭하죠. 폴더를 열고, 파일을 더블클릭하고, 버튼을 누르고.

터미널은 같은 일을 글자를 타이핑해서 하는 거예요.

쉬운 비유

일반 컴퓨터 사용 = 레스토랑에서 메뉴판 보고 손가락으로 가리키기
터미널 = 셰프에게 직접 말로 주문하기
"저기요, 파스타에 버섯 빼고 치즈 두 배로 해주세요!"

터미널은 컴퓨터에게 직접 말(글자)로 명령하는 방법이에요. 더 자유롭고, 더 빠르고, 더 정밀해요.

$ 표시 옆에 명령어를 타이핑하고 엔터를 누르면, 컴퓨터가 결과를 보여줘요. 그게 전부예요!

터미널 여는 법

Windows에서 터미널 열기

방법 1: Win + R 키를 누르고 cmd를 타이핑한 후 엔터
방법 2: 시작 메뉴에서 "터미널" 또는 "Terminal"을 검색
방법 3: 시작 메뉴에서 "PowerShell"을 검색

Mac에서 터미널 열기

방법 1: Cmd + Space로 Spotlight 열고 "Terminal" 타이핑
방법 2: Finder → 응용 프로그램 → 유틸리티 → 터미널

2. Git: 게임의 세이브 포인트

게임할 때 세이브 하잖아요? 보스전 앞에서 "여기 한 번 저장하고..." 하는 거요. 혹시 지면 다시 돌아올 수 있게.

Git은 파일 작업의 세이브 포인트 시스템이에요.

쉬운 비유

Git 없이 작업 = 세이브 없이 게임하기
실수하면 처음부터 다시! 😱

Git으로 작업 = 매 구간마다 세이브하기
실수해도 "마지막 세이브 포인트"로 돌아가면 됩니다!

Git은 여러분이 파일을 수정할 때마다 "이 시점의 상태를 기록해둘게"라고 저장해놓는 도구예요. 나중에 "어, 아까 그 버전이 더 좋았는데?"라고 하면 그때로 돌아갈 수 있어요.

gstack은 작업할 때 Git을 사용해서 변경 사항을 기록해요. 여러분이 직접 Git 명령어를 외울 필요는 없어요. gstack이 알아서 해줍니다. 그냥 "아, 이런 게 있구나" 정도만 알면 돼요.

3. Node.js와 Bun: 코드를 실행하는 엔진

자동차를 타려면 엔진이 있어야 하잖아요? Node.js와 Bun은 코드를 실행하는 엔진이에요.

쉬운 비유

Node.js = 검증된 현대 엔진. 전 세계에서 가장 많이 쓰여요.
Bun = 최신형 터보 엔진. Node.js보다 빠르고 편리해요.

둘 다 JavaScript라는 프로그래밍 언어를 실행하는 엔진이에요. gstack을 설치할 때 둘 중 하나가 필요합니다.

"JavaScript? 프로그래밍 언어?" — 걱정 마세요. 여러분이 직접 코드를 쓸 일은 없어요. 엔진이 뒤에서 조용히 돌아가는 것뿐이에요. 자동차 탈 때 엔진 구조를 몰라도 되는 것처럼요.

정리: 필요한 도구 4가지

gstack을 쓰기 위해 필요한 도구를 정리하면 이렇습니다:

  1. 터미널 — 컴퓨터에 명령어를 타이핑하는 창 (이미 컴퓨터에 있어요!)
  2. Git — 파일 변경 사항을 기록하는 세이브 시스템
  3. Node.js 또는 Bun — 코드를 실행하는 엔진
  4. Claude Code — AI 에이전트 (gstack이 여기서 돌아가요)

실습: 터미널 열고 3개 명령어 쳐보기

위에서 설명한 방법으로 터미널을 열어보세요. 그리고 아래 3개를 하나씩 타이핑하고 엔터를 눌러보세요:

  1. echo "Hello, World!" → 화면에 "Hello, World!"가 나오면 성공!
  2. git --version → "git version 2.xx.x" 같은 게 나오면 Git이 설치된 거예요. "command not found"가 나오면 아직 설치 안 된 거예요. 괜찮아요, 다음 챕터에서 설치해요.
  3. node --version → "v20.x.x" 같은 게 나오면 Node.js가 설치된 거예요. 역시 없어도 괜찮아요!

이 실습의 목적은 "터미널이 무섭지 않다"는 걸 느끼는 것이에요. 명령어를 치고 결과를 확인하는 것, 그게 전부예요!

잠깐, 정리!

  • 터미널은 컴퓨터에 글자로 명령하는 창이에요. 무서워 보이지만 복합기만큼 단순해요.
  • Git은 파일 작업의 세이브 포인트 시스템이에요. 실수해도 돌아갈 수 있어요.
  • Node.js/Bun은 코드를 실행하는 엔진이에요. 뒤에서 조용히 돌아갈 뿐, 여러분이 직접 다룰 일은 없어요.
  • gstack 사용에 필요한 도구는 터미널, Git, Node.js/Bun, Claude Code 4가지예요.
