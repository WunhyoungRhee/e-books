APPENDIX A

용어 사전 (Glossary)

모르는 단어가 나오면 여기로 오세요

이 용어 사전 사용법

이 책을 읽다가 "이게 무슨 뜻이지?" 싶은 단어가 나오면 여기로 돌아오세요. 한국어 가나다순으로 정리해놨어요. 모든 설명은 최대한 쉽게, 일상 언어로 적었어요.

쉬운 비유

이 페이지는 여러분의 "AI 사전"이에요. 영어 공부할 때 모르는 단어 나오면 사전 찾듯이, AI 공부할 때 모르는 단어 나오면 여기를 찾으세요.

용어 목록

AI 에이전트 (AI Agent)

단순히 질문에 답하는 것을 넘어, 스스로 계획을 세우고, 도구를 사용하고, 작업을 실행하는 AI 프로그램이에요. ChatGPT가 "비서"라면, AI 에이전트는 "팀원"에 가까워요. 시키면 알아서 단계를 나눠서 일을 해요.

아키텍처 (Architecture)

소프트웨어의 전체 구조와 설계도예요. 건물을 짓기 전에 설계도를 그리듯, 프로그램을 만들기 전에 어떤 부분이 어떻게 연결되는지 정하는 거예요. "이 앱은 어떤 구조로 만들까?"가 아키텍처 결정이에요.

CLI (Command Line Interface)

글자로 컴퓨터와 대화하는 방식이에요. 마우스 클릭 대신 키보드로 명령어를 입력하는 거예요. gstack은 CLI 도구예요. 까만 화면에 글자를 입력하면 AI가 작업을 수행합니다.

CI (Continuous Integration, 지속적 통합)

여러 사람이 함께 코드를 작성할 때, 코드를 합칠 때마다 자동으로 테스트하는 시스템이에요. 여러 명이 같은 문서를 수정할 때, 저장할 때마다 자동으로 맞춤법 검사가 돌아간다고 생각하면 돼요.

디버깅 (Debugging)

프로그램에서 오류(버그)를 찾아서 고치는 과정이에요. "디버그"는 "벌레(bug)를 제거한다(de-)"는 뜻이에요. 옛날에 진짜 나방이 컴퓨터 안에 들어가서 고장을 일으킨 데서 유래했어요.

엣지케이스 (Edge Case)

보통은 안 일어나지만, 가끔 발생하는 특수한 상황이에요. 예를 들어, 이름 입력 칸에 이모지를 넣거나, 생년월일에 미래 날짜를 넣는 것처럼 "설마 이렇게 하겠어?"라는 상황이 엣지케이스예요.

Git

파일의 변경 이력을 추적하는 도구예요. "저장" 버튼을 누를 때마다 이전 버전이 기록되어서, 언제든 과거로 돌아갈 수 있어요. 게임의 세이브 포인트 같은 거예요. 개발자들이 협업할 때 필수로 사용해요.

생성형 AI (Generative AI)

새로운 콘텐츠를 만들어내는 AI예요. 글, 코드, 이미지, 음악 등을 "없던 것"에서 새로 만들어요. "다음에 올 단어를 예측하는" 원리로 작동하는데, 그 예측 능력이 엄청나게 뛰어나서 사람처럼 자연스러운 결과물을 만들어냅니다.

머지 (Merge)

여러 사람이 각자 수정한 코드를 하나로 합치는 것이에요. 학교 조별 과제에서 각자 맡은 부분을 하나의 문서로 합치는 것과 같아요. 가끔 같은 부분을 서로 다르게 수정하면 "충돌"이 발생하기도 해요.

MVP (Minimum Viable Product, 최소 기능 제품)

제품의 핵심 기능만 넣어서 가장 빠르게 만들어 출시하는 버전이에요. 100가지 기능이 있는 완벽한 앱 대신, 핵심 기능 3개만 넣은 앱을 먼저 내놓고 사용자 반응을 보는 거예요. 스타트업의 생존 전략이에요.

OWASP

웹 보안 취약점을 연구하고 가이드를 제공하는 국제 단체예요. "OWASP Top 10"은 가장 흔한 보안 취약점 10가지 목록이에요. gstack의 /cso 스킬이 이 목록을 기준으로 보안 점검을 해줘요.

PR (Pull Request)

"내가 수정한 코드를 합쳐주세요"라고 요청하는 것이에요. 조별 과제에서 "내 부분 다 썼으니 확인하고 합쳐주세요"라고 하는 것과 같아요. 다른 사람이 코드를 검토(리뷰)한 후 합쳐요.

프로덕션 (Production)

실제 사용자가 쓰는 진짜 서비스 환경이에요. 개발하고 테스트하는 환경과 구분돼요. "프로덕션에 배포한다"는 건 "진짜 사용자한테 공개한다"는 뜻이에요. 여기서 문제가 생기면 실제 사용자가 영향을 받아요.

프롬프트 (Prompt)

AI에게 보내는 메시지, 요청, 지시예요. 카카오톡 메시지를 보내듯 AI에게 하고 싶은 말을 적는 거예요. "이메일 써줘"도 프롬프트이고, "이 코드의 버그를 찾아줘"도 프롬프트예요.

QA (Quality Assurance, 품질 보증)

소프트웨어가 제대로 작동하는지 검사하는 과정이에요. 공장에서 제품을 출하하기 전에 불량품을 걸러내는 것처럼, 서비스를 출시하기 전에 버그나 문제가 없는지 확인하는 거예요. gstack의 /qa 스킬이 이걸 자동으로 해줘요.

스킬 (Skill)

AI에게 특정 작업을 시키는 레시피(SKILL.md 파일)예요. 요리 레시피처럼 "이런 재료로 이렇게 만들어"라고 정해놓으면, AI가 그대로 따라해요. gstack에는 28개 이상의 기본 스킬이 있고, 직접 만들 수도 있어요.

스프린트 (Sprint)

정해진 짧은 기간(보통 1~2주) 동안 집중해서 작업하는 단위예요. 마라톤처럼 길게 달리는 대신, 100미터씩 전력 질주하고 쉬고를 반복하는 방식이에요. "이번 스프린트에서는 로그인 기능을 완성하자" 같은 식으로 목표를 정해요.

SRE (Site Reliability Engineering, 사이트 신뢰성 엔지니어링)

서비스가 24시간 안정적으로 돌아가도록 관리하는 분야예요. 건물 관리인이 엘리베이터, 전기, 수도를 관리하듯, SRE 엔지니어는 서버, 네트워크, 데이터베이스가 잘 작동하도록 관리해요.

STRIDE

보안 위협을 6가지 카테고리로 분류하는 분석 방법이에요. Spoofing(위장), Tampering(변조), Repudiation(부인), Information Disclosure(정보 유출), Denial of Service(서비스 거부), Elevation of Privilege(권한 상승)의 약자예요. /cso 스킬이 이 방법론을 사용해요.

터미널 (Terminal)

컴퓨터에게 글자로 명령을 내리는 프로그램이에요. 까만 배경에 하얀 글자가 나오는 그 화면이요. Windows에서는 PowerShell이나 명령 프롬프트, Mac에서는 터미널 앱이에요. gstack은 이 터미널 안에서 작동합니다.

용어가 어려워도 괜찮아요

처음에는 이 용어들이 외계어처럼 느껴질 수 있어요. 완전 정상이에요.

중요한 건 한 번에 다 외우려고 하지 않는 것이에요. gstack을 쓰다가 모르는 단어가 나오면 여기 와서 찾아보고, 다시 돌아가서 계속하면 돼요. 몇 번 반복하다 보면 자연스럽게 익숙해져요.
