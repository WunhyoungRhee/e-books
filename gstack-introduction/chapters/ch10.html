CHAPTER 10

학습자를 위한 가이드

gstack은 최고의 AI 교과서예요

유튜브로 공부해본 적 있죠?

"AI 배우기"를 검색하면 영상이 10만 개 나와요. 그중에 어떤 걸 봐야 할지 모르겠고, 영상 하나 보면 또 모르는 단어가 나와서 그걸 또 검색하고... 3시간 후엔 고양이 영상을 보고 있는 자신을 발견하죠.

gstack이 특별한 이유는 쓰면서 배우는 구조이기 때문이에요. 영상을 보는 게 아니라, 직접 해보면서 원리를 깨달아요. 자전거를 유튜브로 배울 수 없듯이, AI도 직접 타봐야 해요.

쉬운 비유

유튜브로 수영 배우기 = 이론은 알지만 물에 들어가면 허우적
gstack으로 배우기 = 옆에 코치가 있는 수영장에서 직접 연습

gstack으로 배울 수 있는 4가지

1. 에이전트 워크플로우 (Agent Workflow)

AI가 단순히 질문에 답하는 게 아니라, 여러 단계를 스스로 계획하고 실행하는 방식을 배워요. "이메일 써줘"는 기본이고, "프로젝트를 분석하고 → 계획을 세우고 → 실행하고 → 검증하는" 전체 흐름을 이해하게 돼요.

2. SKILL.md 작성법

AI에게 특정 작업을 시키는 레시피(SKILL.md)를 직접 읽고, 수정하고, 만들어보면서 "AI에게 정확한 지시를 내리는 법"을 체득해요. 이건 어떤 AI 도구를 쓰든 평생 써먹을 수 있는 능력이에요.

3. 브라우저 자동화

gstack의 /browse 스킬로 웹사이트를 자동으로 테스트하는 법을 배워요. 클릭하고, 스크롤하고, 스크린샷 찍고, 에러를 확인하는 과정을 AI가 알아서 해주는 걸 보면서 "자동화"의 힘을 실감하게 돼요.

4. 프롬프팅 실전 기술

gstack을 쓰다 보면 자연스럽게 프롬프트를 잘 쓰게 돼요. "이렇게 말하면 AI가 이렇게 반응하는구나"를 수백 번 경험하면서, 어떤 AI 도구든 능숙하게 다루는 사람이 됩니다.

4단계 학습 경로

어디서부터 시작해야 할지 모르겠다면, 이 순서를 따라가세요.

단계 목표 해볼 것 예상 시간
Lv.1 구경하기 gstack이 뭔지 감 잡기 ch01~ch05 읽기, 설치해보기 2시간
Lv.2 따라하기 기본 스킬 직접 실행 /browse, /qa 써보기 3시간
Lv.3 이해하기 SKILL.md 구조 파악 기존 스킬 분석, 수정해보기 5시간
Lv.4 만들기 나만의 스킬 제작 새 SKILL.md 작성, 테스트, 공유 8시간

서두르지 마세요

Lv.1에서 Lv.4까지 한 번에 달리려고 하면 지쳐요. 하루에 한 단계씩, 일주일이면 충분해요. 중요한 건 매일 조금씩 만져보는 것이에요.

꼭 알아야 할 4가지 개념

gstack을 쓰다 보면 반복적으로 만나는 핵심 개념이 있어요. 이것만 이해하면 나머지는 자연스럽게 따라옵니다.

1. 레퍼런스 시스템 (Reference System)

gstack의 스킬들은 서로 연결되어 있어요. /conductor가 계획을 세우면, /browse가 테스트하고, /cso가 보안을 점검해요. 마치 회사에서 팀원들이 각자 역할을 맡아 협업하는 것처럼요. 스킬 A의 결과물이 스킬 B의 입력이 되는 이 흐름을 "레퍼런스 시스템"이라고 해요.

2. 데몬 모델 (Daemon Model)

gstack의 /browse는 백그라운드에서 브라우저를 조용히 돌려요. 여러분이 다른 작업을 하는 동안에도 웹사이트를 모니터링하거나 테스트할 수 있죠. 이걸 "데몬(Daemon)"이라고 해요. 눈에 보이지 않지만 묵묵히 일하는 도우미라고 생각하면 돼요.

쉬운 비유

데몬 = 로봇 청소기. 여러분이 외출한 사이에 알아서 바닥을 청소해주잖아요? 데몬도 비슷해요. 백그라운드에서 알아서 작업을 수행합니다.

3. 아티팩트 플로우 (Artifact Flow)

gstack이 작업하면 결과물(아티팩트)이 생겨요. 코드 파일, 보고서, 스크린샷 같은 것들이요. 이 결과물들이 다음 단계의 입력으로 자연스럽게 흘러가는 걸 "아티팩트 플로우"라고 해요. 공장의 컨베이어 벨트를 상상하면 딱이에요.

4. 리뷰 대시보드 (Review Dashboard)

작업 결과를 한눈에 확인하는 화면이에요. /qa가 찾은 버그, /cso가 발견한 보안 이슈, /design-review가 지적한 디자인 문제 — 이 모든 걸 정리된 형태로 볼 수 있어요. 학교 성적표처럼 현재 프로젝트의 "건강 상태"를 보여주는 거예요.

직접 해보기: 4가지 실습

실습 1: 스킬 목록 탐험하기

gstack을 실행하고 /find-skills을 입력해보세요. 어떤 스킬들이 있는지 쭉 훑어보고, 가장 궁금한 3개를 메모하세요.

실습 2: SKILL.md 읽어보기

아무 스킬 하나를 골라서 SKILL.md 파일을 열어보세요. 어떤 구조로 되어 있는지 관찰하세요. 아직 완벽히 이해 못 해도 괜찮아요. "아, 이런 형식이구나" 정도면 충분해요.

실습 3: /browse로 웹사이트 탐험하기

/browse 스킬로 자주 방문하는 웹사이트를 열어보세요. 스크린샷을 찍고, 특정 버튼을 클릭해보세요. AI가 브라우저를 조작하는 모습이 신기할 거예요.

실습 4: 프롬프트 실험하기

같은 요청을 다른 방식으로 3번 해보세요. 예를 들어 "코드 리뷰해줘", "이 코드에서 버그 찾아줘", "시니어 개발자 관점에서 이 코드 평가해줘" — 프롬프트에 따라 결과가 어떻게 달라지는지 비교해보세요.

잠깐, 정리!

  • gstack으로 에이전트 워크플로우, SKILL.md, 브라우저 자동화, 프롬프팅 4가지를 배울 수 있어요
  • 4단계 학습 경로: 구경 → 따라하기 → 이해하기 → 만들기
  • 핵심 개념 4가지: 레퍼런스 시스템, 데몬 모델, 아티팩트 플로우, 리뷰 대시보드
  • 가장 좋은 학습 방법은 직접 해보는 것. 유튜브 10시간보다 실습 1시간이 낫습니다
