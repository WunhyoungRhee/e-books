CHAPTER 1

AI와 대화하기: ChatGPT에서 한 걸음 더

여러분은 이미 AI를 쓰고 있어요. 다만 몰랐을 뿐

카카오톡에서 시작된 이야기

카카오톡에서 "오늘 점심 뭐 먹지?" 물어본 적 있죠? 친구한테요. 그런데 혹시 ChatGPT에도 물어봤다면 — 축하해요. 여러분은 이미 AI 사용자입니다.

"에이, 그건 그냥 검색 같은 거 아니에요?"

아니에요. 여러분은 그 순간, AI에게 질문을 던지고, AI가 새로운 답변을 만들어내는 경험을 한 거예요. 구글 검색은 이미 있는 페이지를 찾아주지만, ChatGPT는 여러분의 질문에 맞춰 그 자리에서 새로운 글을 만들어냅니다. 완전히 다른 거예요.

이 챕터에서는 여러분이 이미 알고 있는 것(ChatGPT)에서 출발해서, "아, AI가 이런 것도 할 수 있구나!"까지 한 걸음 더 나아가볼 거예요.

생성형 AI, 한 문장으로

여러분 핸드폰 키보드 아시죠? 카카오톡에서 글자를 치면, 그 위에 다음에 올 단어를 추천해주는 기능이요. "오늘 점심"이라고 치면 "뭐 먹을까?"가 뜨는 거.

생성형 AI(Generative AI)는 그 기능의 초울트라 버전이에요. 핸드폰 키보드는 여러분이 자주 쓰는 단어 몇백 개를 기억하는 수준이지만, 생성형 AI는 인터넷에 있는 수십억 개의 글, 책, 논문을 학습해서 "이 맥락에서 다음에 올 말은 뭘까?"를 예측합니다.

쉬운 비유

핸드폰 키보드 추천 = 동네 편의점 삼각김밥
생성형 AI = 미슐랭 셰프가 그 자리에서 만들어주는 맞춤 요리

둘 다 "다음에 뭐가 올까?"를 예측하는 건데, 규모와 정밀함이 차원이 달라요.

그래서 "오늘 점심 뭐 먹지? 나 매운 거 좋아하고 강남역 근처야"라고 물으면, AI가 여러분의 취향, 위치, 시간대까지 고려해서 맞춤 추천을 만들어줍니다. 검색 결과를 보여주는 게 아니라, 새로운 답변을 생성하는 거예요.

프롬프트? 이미 하고 있었어요

"프롬프트(Prompt)"라는 단어, 좀 무섭게 들릴 수 있어요. 하지만 사실 여러분은 이미 매일 하고 있어요.

프롬프트(Prompt)란?

AI에게 보내는 메시지. 그게 다예요. ChatGPT 채팅창에 "이메일 써줘"라고 타이핑한 게 프롬프트예요. 카톡 보내는 거랑 똑같아요.

혹시 ChatGPT에 이런 거 물어본 적 있나요?

  • "이 영어 문장 번역해줘"
  • "면접 준비 도와줘"
  • "맛집 추천해줘"
  • "이메일 좀 다듬어줘"

이 하나하나가 전부 프롬프트예요. 여러분은 이미 프롬프팅(Prompting)을 하고 있었던 거예요. 그냥 이름을 몰랐을 뿐!

ChatGPT vs Claude Code: 조언하는 친구 vs 직접 해주는 동료

여기서 재미있는 비교를 하나 해볼게요. ChatGPT와 Claude Code는 뭐가 다를까요?

쉬운 비유

ChatGPT = 조언을 잘해주는 친구
"이사할 때 이렇게 하면 좋아!" 하고 알려줘요.

Claude Code = 직접 와서 짐을 옮겨주는 동료
"그래, 이 박스부터 옮길게. 다음 뭐 할까?" 하면서 실제로 일을 해요.

ChatGPT는 브라우저 채팅창에서 대화하면서 답변을 보여주는 도구예요. "이메일 써줘"하면 이메일을 보여주고, 여러분이 복사-붙여넣기를 해야 해요.

Claude Code는 여러분의 컴퓨터에서 직접 파일을 만들고, 수정하고, 실행하는 도구예요. "이메일 써줘"가 아니라 "이 프로젝트 전체를 만들어줘"가 가능해요.

정리하면, ChatGPT는 "뭘 해야 하는지 알려주는" 도구이고, Claude Code는 "직접 해주는" 도구예요. 둘 다 훌륭한 도구인데, 하는 역할이 달라요.

에이전트(Agent)란? 레시피 블로거 vs 진짜 셰프

여기서 중요한 개념이 하나 나와요. 에이전트(Agent).

쉬운 비유

일반 AI (ChatGPT) = 레시피 블로거
"감자탕 이렇게 만드세요!" 하고 레시피를 알려줘요. 맛있어 보이지만, 요리는 여러분이 직접 해야 해요.

에이전트 (Claude Code) = 진짜 셰프
여러분 집 부엌에 와서 직접 감자탕을 끓여줘요. 재료도 확인하고, 간도 봐가면서, 완성된 요리를 내놓아요.

에이전트는 단순히 답변을 생성하는 것을 넘어서, 여러 단계의 작업을 스스로 계획하고 실행하는 AI예요. "A 하고, B 하고, C 한 다음에 결과 보여줘"라고 하면, 진짜로 A → B → C를 순서대로 해냅니다.

Claude Code가 바로 이런 에이전트예요. 여러분의 컴퓨터에서 파일을 읽고, 코드를 쓰고, 명령어를 실행하면서 작업을 완성해요.

스킬(Skill)이란? 셰프의 레시피북

에이전트가 셰프라면, 스킬(Skill)은 셰프의 레시피북이에요.

아무리 뛰어난 셰프라도 매번 새로운 요리를 즉흥으로 만들면 시간이 오래 걸리죠. 하지만 검증된 레시피북이 있으면? "아, 이 요리는 이 레시피 따라가면 돼" 하면서 빠르고 정확하게 만들어요.

스킬도 마찬가지예요. Claude Code에게 "이런 상황에서는 이렇게 일해" 하고 미리 적어둔 매뉴얼이에요.

스킬(Skill) = AI를 위한 업무 매뉴얼

예를 들어 "코드 리뷰 스킬"이 있으면, Claude Code는 코드를 검토할 때 어떤 기준으로 봐야 하는지, 어떤 순서로 확인해야 하는지 미리 알고 있어요. 매번 설명할 필요가 없죠.

스킬이 많을수록 셰프가 더 다양한 요리를 잘 만들 수 있듯이, Claude Code도 더 다양한 작업을 잘 해내게 됩니다.

전체 그림: GenAI → Claude Code → gstack

자, 이제 전체 구조를 한눈에 볼까요?

생성형 AI는 두뇌예요. 말을 이해하고 생성하는 능력.

Claude Code는 두뇌에 손발을 붙인 거예요. 실제로 컴퓨터에서 작업을 수행하는 에이전트.

gstack은 그 에이전트에게 28가지 전문 기술을 가르친 거예요. 마치 신입 직원에게 업무 매뉴얼 28권을 준 것처럼!

슬래시 명령어: AI에게 주문 걸기

gstack의 스킬을 사용하는 방법은 아주 간단해요. 슬래시 명령어(Slash Command)를 쓰면 돼요.

카카오톡에서 "/투표"를 치면 투표 기능이 뜨는 것처럼, Claude Code에서 /office-hours를 치면 "오피스 아워" 스킬이 실행됩니다.

슬래시 명령어란?

/로 시작하는 특별한 명령어예요. /office-hours, /ship, /review 같은 것들이요. 각각이 하나의 스킬을 실행시키는 주문(呪文) 같은 거예요.

예를 들어:

  • /office-hours → 아이디어를 정리하고 검증하는 스킬
  • /plan-ceo-review → 만든 계획을 CEO 관점에서 검토하는 스킬
  • /ship → 완성된 코드를 배포하는 스킬

이런 슬래시 명령어 28개가 모여 있는 게 바로 gstack이에요. 처음엔 "28개나?!" 할 수 있지만, 걱정 마세요. 일단 5개만 알면 충분해요. 나머지는 필요할 때 하나씩 배우면 됩니다.

직접 해보기

실습: ChatGPT와 gstack, 뭐가 다를까?

ChatGPT(또는 다른 AI 챗봇)를 열고, 이렇게 물어보세요:

"동네 카페 추천 웹사이트를 만들고 싶어. 어떻게 해야 해?"

ChatGPT가 답변을 줄 거예요. 아마 "먼저 이런 기술을 배우세요, 그 다음 이렇게 하세요..." 같은 조언을 해줄 거예요.

이 책을 다 읽고 gstack을 설치하면, 같은 질문을 Claude Code에 하면 됩니다. 그러면 AI가 조언을 하는 게 아니라 직접 웹사이트를 만들어줄 거예요. 이 차이를 기억해두세요!

잠깐, 정리!

  • 생성형 AI는 핸드폰 키보드 추천의 초울트라 버전이에요. "다음에 올 말"을 예측해서 새로운 글을 만들어내요.
  • 프롬프트는 AI에게 보내는 메시지예요. ChatGPT에 질문한 적 있다면, 이미 프롬프팅을 해본 거예요.
  • ChatGPT는 조언해주는 친구, Claude Code는 직접 일해주는 동료예요.
  • 에이전트는 여러 단계를 스스로 계획하고 실행하는 AI이고, 스킬은 에이전트를 위한 업무 매뉴얼이에요.
  • gstack은 28개의 스킬이 모여 있는 패키지예요. 슬래시 명령어로 사용해요.
