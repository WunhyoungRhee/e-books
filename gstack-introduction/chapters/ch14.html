CHAPTER 14

참고 자료

더 알고 싶을 때 찾아보세요

이 책이 끝이 아니에요

여기까지 읽으셨다면 정말 대단해요! 근데 이 책은 시작점이지, 끝이 아니에요. AI와 에이전트의 세계는 매일 새로운 것들이 나오고 있어요.

여기에 여러분의 다음 걸음을 도와줄 자료들을 모아놨어요. 전부 볼 필요는 없어요. 관심 가는 것부터 하나씩 살펴보세요.

초보자를 위한 자료 (여기서 시작하세요!)

"너무 많아서 뭘 봐야 할지 모르겠다"는 분들을 위해, 우선순위를 매겨놨어요.

1순위: 이것부터 보세요

  • Claude Code 공식 튜토리얼 — 설치부터 첫 프로젝트까지 단계별 안내. 이 책의 ch03~ch05를 더 깊이 다루는 버전이에요.
  • gstack 스킬 갤러리 — 어떤 스킬들이 있는지 구경할 수 있어요. "아, 이런 것도 가능하구나!"를 느끼기 좋아요.

2순위: 실력이 붙으면 보세요

  • SKILL.md 작성 가이드 — 나만의 스킬을 만들고 싶을 때 참고하세요.
  • 프롬프트 엔지니어링 가이드 — AI에게 더 정확하게 지시하는 방법을 배울 수 있어요.

3순위: 전문가 레벨

  • Claude API 문서 — 직접 API를 호출해서 자동화 시스템을 구축하고 싶을 때.
  • 에이전트 아키텍처 가이드 — AI 에이전트가 내부적으로 어떻게 작동하는지 깊이 이해하고 싶을 때.

쉬운 비유

운전면허로 치면:
1순위 = 교통 표지판 익히기 (기본)
2순위 = 고속도로 합류 연습 (중급)
3순위 = 서킷 트랙 주행 (고급)

1순위만으로도 충분히 잘 달릴 수 있어요!

공식 문서

  • Anthropic 공식 문서 — docs.anthropic.com
    Claude의 모든 기능을 다루는 공식 문서예요. API 레퍼런스, 모델 사양, 사용 가이드가 있어요.
  • Claude Code 문서 — docs.anthropic.com/claude-code
    Claude Code의 설치, 설정, 사용법을 다루는 공식 가이드예요.
  • Anthropic 쿡북 — github.com/anthropics/anthropic-cookbook
    실제 사용 사례와 코드 예제가 가득한 저장소예요. "이런 걸 어떻게 만들지?" 할 때 여기를 찾아보세요.

튜토리얼과 아티클

  • 프롬프트 엔지니어링 가이드 — Anthropic 프롬프트 가이드
    AI에게 효과적으로 지시하는 방법을 체계적으로 배울 수 있어요. 초보자부터 고급 사용자까지 모두에게 유용해요.
  • "AI 에이전트란 무엇인가?" — AI 에이전트의 개념과 작동 원리를 쉽게 설명하는 글이에요. 이 책의 내용을 복습하고 싶을 때 좋아요.
  • OWASP Top 10 — owasp.org
    웹 애플리케이션 보안의 가장 흔한 취약점 10가지. /cso 스킬이 무엇을 검사하는지 이해하고 싶을 때 참고하세요.
  • Git 입문 가이드 — git-scm.com/book/ko
    Git을 한국어로 배울 수 있는 공식 가이드. ch03에서 소개한 Git의 기초를 더 깊이 배우고 싶을 때.

커뮤니티

혼자 배우는 것보다 함께 배우는 게 훨씬 재밌고 효과적이에요. 이런 곳에서 동료를 찾아보세요.

  • Anthropic Discord
    Claude 사용자들이 모여서 팁을 공유하고, 질문하고, 서로 도와주는 커뮤니티예요. 초보자 질문도 환영해요.
  • GitHub Discussions
    gstack 관련 기술적 토론이 이루어지는 곳이에요. 스킬 제작 팁, 버그 제보, 기능 요청을 할 수 있어요.
  • 한국 AI 커뮤니티
    한국어로 AI를 배우고 토론할 수 있는 커뮤니티들이 있어요. 네이버 카페, 카카오 오픈채팅방 등에서 "Claude Code", "AI 에이전트" 키워드로 검색해보세요.

커뮤니티 참여 팁

"초보라서 질문하기 부끄럽다"는 생각 버리세요. 모든 전문가도 처음엔 초보였어요. 그리고 잘 정리된 초보 질문은 다른 초보자들에게도 큰 도움이 돼요. 여러분의 질문이 누군가의 답이 될 수 있습니다.

추천 학습 경로

"그래서 뭘 어떤 순서로 하라는 거예요?" 하시는 분들을 위해:

  1. 이 책 다시 읽기: 한 번 읽고 끝내지 말고, 실습하면서 다시 읽어보세요. 두 번째 읽으면 보이는 게 달라져요.
  2. 매일 gstack 10분씩 써보기: 거창한 프로젝트가 아니어도 돼요. 간단한 질문이라도 매일 해보세요.
  3. 스킬 하나 분석하기: 관심 가는 스킬의 SKILL.md를 열어서 구조를 이해해보세요.
  4. 나만의 스킬 만들기: 작은 것부터 시작하세요. "인사말 만들어주는 스킬" 같은 것도 훌륭한 첫 프로젝트예요.
  5. 커뮤니티 참여하기: 만든 스킬을 공유하고 피드백을 받아보세요.
